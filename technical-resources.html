Technical Resources

Frequencies & Legal Use

  • 915 MHz ISM (US): license-free (FCC Part 15)
  • Ham bands (70cm): experimental use requires proper licensing/config
  • Power/antennas: follow FCC rules for your license and band

Antennas & Range

  • ¼-wave whips, dipoles, or directional (Yagi/panel) for long links
  • Elevation is king; avoid metal/obstructions; use quality coax

Node Types

  • Personal: portable, battery-powered
  • Router: fixed site, good antenna, relays traffic
  • Solar: remote 24/7 nodes for gaps

Mapping & Planning

  • Meshtastic community maps for node visibility
  • Radio Mobile / QRadioPredict for RF modeling
  • CNYmesh coverage map for our six focus counties
  • CNYmesh Live Dashboard: Real-time MQTT data and mesh visualization

Best Practices for CNY

  • Weather-proof outdoor installs (snow/ice/cold)
  • Dense nodes in Syracuse; high-elevation routers in rural counties
  • Plan long-range bridges toward Rochester, Buffalo, Ithaca, Albany

MeshCore Repeater Settings

CNYmesh is publishing common MeshCore repeater settings so operators across Central New York are using the same baseline configuration. Please update repeaters and fixed infrastructure nodes to match the current CNY MeshCore settings below.

Base Radio Configuration

  • Frequency 910.525 MHz
  • Bandwidth 62.5 kHz
  • Spreading Factor 7
  • Coding Rate 8
  • Transmit Power 22

Apply Base Settings

  • set freq 910525
  • set radio bw 62500 sf 7 cr 8
  • set tx 22

Verify Base Settings

  • get radio
  • get freq
  • get tx

2-Byte Hash Mode

MeshCore repeaters should use 2-byte hash mode. This helps reduce ID collisions and improves overall mesh reliability as the regional network grows.

Apply 2-Byte Hash Mode and Sync Clock

  • set path.hash.mode 1
  • reboot
  • clock sync

Verify Hash Mode After Reboot

  • get path.hash.mode

After rebooting, also check the device status tab to confirm uptime and clock status.

Regional Configuration

CNYmesh uses a nested regional structure so traffic can be organized without creating unnecessary flood traffic for neighboring meshes.

  • us — national level
  • us-ny — New York State level
  • us-ny-cny — Central New York local level

Regional or statewide channels should use us-ny where appropriate. Local CNY repeater and flood traffic should use the CNY regional structure so local traffic does not unnecessarily propagate into neighboring local meshes.

Apply Repeater Regions

  • region put us
  • region put us-ny us
  • region put us-ny-cny us-ny
  • region allowf us
  • region allowf us-ny
  • region allowf us-ny-cny
  • region save

Verify Regional Settings

  • region

The region output should show the nested structure from us to us-ny to us-ny-cny.
