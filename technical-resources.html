Technical Resources

Frequencies & Legal Use

  • 915 MHz ISM (US): license-free (FCC Part 15)
  • Ham bands (70cm): experimental use requires proper licensing/config
  • Power/antennas: follow FCC rules for your license and band

Antennas & Range

  • ¼-wave whips, dipoles, or directional (Yagi/panel) for long links
  • Elevation is king; avoid metal/obstructions; use quality coax

Node Types

  • Personal: portable, battery-powered
  • Router: fixed site, good antenna, relays traffic
  • Solar: remote 24/7 nodes for gaps

Mapping & Planning

  • Meshtastic community maps for node visibility
  • Radio Mobile / QRadioPredict for RF modeling
  • CNYmesh coverage map for our six focus counties
  • CNYmesh Live Dashboard: Real-time MQTT data and mesh visualization

Best Practices for CNY

  • Weather-proof outdoor installs (snow/ice/cold)
  • Dense nodes in Syracuse; high-elevation routers in rural counties
  • Plan long-range bridges toward Rochester, Buffalo, Ithaca, Albany

Meshtastic Infrastructure Node Settings

Infrastructure nodes are fixed, powered Meshtastic nodes placed at high-elevation or strategic locations to extend CNYmesh coverage. Follow the settings below to configure your node as a reliable infrastructure relay.

Choosing the Correct Node Role

Meshtastic offers several node roles. Choosing the right one is important for infrastructure nodes.

  • Router ROUTER — Recommended for most infrastructure nodes. Prioritizes packet relay over client features. Sends position and telemetry at reduced intervals, maximizes hop participation, and remains fully manageable over the mesh.
  • Router and Client ROUTER_CLIENT — A good choice if the operator also wants to use the node for messaging. Behaves like Router but retains full client functionality.
  • Repeater REPEATER — Often the wrong choice. Repeater disables position broadcasts, telemetry, admin-over-mesh, and most user-facing features. It is designed only for bare-minimum relay hardware where memory and CPU are severely constrained. Avoid using Repeater unless you have a specific reason and understand the limitations.
  • Client CLIENT — Default mode. Suitable for personal devices but not optimized for infrastructure use.

Summary: Set your infrastructure node to ROUTER or ROUTER_CLIENT. Reserve REPEATER only for dedicated bare-relay hardware where features are intentionally not needed.

LoRa Radio Configuration

  • Use Modem Preset: True
  • Modem Preset: Long_Fast
  • Region: United States
  • Hop Limit: 3 (minimum) - 7 (maximum)
  • TX Enabled: True
  • TX Power: 30
  • Frequency Slot: 20
  • Override Duty Cycle: True
  • Override Frequency: 906.875 MHz
  • Ignore MQTT: False
  • OK to MQTT: True

MQTT Configuration

Enable MQTT on infrastructure nodes to bridge the local mesh to the regional network and allow remote monitoring via the CNYmesh Live Dashboard.

  • MQTT Server: mqtt.meshtastic.org
  • Root Topic: msh/US/NY/CNY
  • Uplink Enabled: True (on CNYmesh channel)
  • Downlink Enabled: True (on CNYmesh channel)

Power & Uptime Recommendations

  • Power infrastructure nodes from AC or solar for continuous 24/7 operation
  • Disable power-saving / sleep modes to ensure the node always participates in routing
  • Place nodes at elevation — rooftops, towers, or hilltops — to maximize range
  • Use quality weatherproof enclosures for outdoor installs (critical for CNY winters)

MeshCore Repeater Settings

CNYmesh is publishing common MeshCore repeater settings so operators across Central New York are using the same baseline configuration. Please update repeaters and fixed infrastructure nodes to match the current CNY MeshCore settings below.

Base Radio Configuration

  • Frequency 910.525 MHz
  • Bandwidth 62.5 kHz
  • Spreading Factor 7
  • Coding Rate 8
  • Transmit Power 22

Apply Base Settings

  • set freq 910525
  • set radio bw 62500 sf 7 cr 8
  • set tx 22

Verify Base Settings

  • get radio
  • get freq
  • get tx

2-Byte Hash Mode

MeshCore repeaters should use 2-byte hash mode. This helps reduce ID collisions and improves overall mesh reliability as the regional network grows.

Apply 2-Byte Hash Mode and Sync Clock

  • set path.hash.mode 1
  • reboot
  • clock sync

Verify Hash Mode After Reboot

  • get path.hash.mode

After rebooting, also check the device status tab to confirm uptime and clock status.

Regional Configuration

CNYmesh uses a nested regional structure so traffic can be organized without creating unnecessary flood traffic for neighboring meshes.

  • us — national level
  • us-ny — New York State level
  • us-ny-cny — Central New York local level

Regional or statewide channels should use us-ny where appropriate. Local CNY repeater and flood traffic should use the CNY regional structure so local traffic does not unnecessarily propagate into neighboring local meshes.

Apply Repeater Regions

  • region put us
  • region put us-ny us
  • region put us-ny-cny us-ny
  • region allowf us
  • region allowf us-ny
  • region allowf us-ny-cny
  • region save

Verify Regional Settings

  • region

The region output should show the nested structure from us to us-ny to us-ny-cny.
