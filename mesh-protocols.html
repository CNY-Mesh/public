Mesh Protocols

CNYmesh actively operates networks on both Meshtastic and MeshCore. This page explains what each protocol does, how they differ, and which might be the right starting point for you.

Two protocols, one community

Both Meshtastic and MeshCore use LoRa radios to build decentralized mesh networks that can carry text messages, position reports, and telemetry without relying on the internet or cell service. They share the same underlying radio technology but take different approaches to how packets are routed and how the network is organized.

CNYmesh doesn't pick sides. We run both because each has genuine strengths, and having parallel networks gives our community more flexibility, more to learn from, and better long-term resilience.

Meshtastic

Meshtastic is a mature, widely-adopted open-source mesh platform with a large global community. It uses a managed flood routing model — messages propagate hop-by-hop through nearby nodes until they reach their destination or hit the hop limit. Configuration is done through polished mobile and desktop apps with no command line required.

Strengths

  • Large, established ecosystem with broad hardware support
  • Polished mobile apps for Android and iOS
  • Low barrier to entry — most users are up and running in under an hour
  • Active global community with lots of documentation and peer support
  • MQTT bridging for internet-connected mesh expansion

Things to consider

  • Flood routing can create channel congestion on dense networks
  • Limited control over routing topology at the protocol level

MeshCore

MeshCore is a newer open-source LoRa mesh firmware with a focus on scalable, hierarchical routing. Rather than flooding packets across the network, MeshCore builds and maintains path information so that traffic can be routed more efficiently over larger areas. It supports a nested regional structure that's well-suited for multi-city coordination.

Strengths

  • Path-aware routing reduces unnecessary retransmissions
  • Designed to scale across regional and multi-regional networks
  • Explicit repeater and room-server roles for structured deployments
  • Active development with growing momentum in the community mesh space

Things to consider

  • Smaller ecosystem and hardware options compared to Meshtastic
  • Initial configuration requires more hands-on setup
  • Still maturing — expect more frequent changes than Meshtastic

Which should I start with?

Start with Meshtastic if…

  • You want the quickest path to a working node
  • You prefer configuring everything through a smartphone app
  • You want the widest choice of off-the-shelf hardware
  • You plan to use MQTT to monitor your node remotely
  • You're new to mesh networking

Meshtastic guide

Start with MeshCore if…

  • You're interested in the technical side of mesh routing
  • You want to contribute to a newer, evolving ecosystem
  • You're planning a fixed infrastructure deployment
  • You like the idea of a regionally-structured network
  • You already have Meshtastic running and want to explore more

MeshCore guide

You don't have to choose just one

Many CNYmesh members run both. Some hardware can even run Meshtastic and MeshCore on separate boards deployed at the same location. CNYmesh actively documents what we learn from running both in parallel — join the community to follow along.
