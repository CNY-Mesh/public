Getting Started with MeshCore

A practical guide to getting a MeshCore node running and connected to the CNYmesh network — covering hardware, firmware, the companion app, and CNY-specific radio settings.

What is MeshCore?

MeshCore is an open-source LoRa mesh networking firmware with a different architectural approach than Meshtastic. It uses a store-and-forward routing model with explicit path tracking, which makes it well-suited for building regional hierarchical networks. CNYmesh is actively deploying MeshCore infrastructure alongside its Meshtastic network.

Not sure which protocol is right for you? See the Mesh Protocols comparison page.

Step 1 — Get compatible hardware

MeshCore runs on several popular LoRa platforms. Look for boards using the SX1262 or SX1276 radio chipset in the 915 MHz band for US use. Common choices include:

  • Heltec LoRa 32 V3 — compact, integrated display, widely available
  • LILYGO T-Beam — GPS-equipped, good for mobile or infrastructure use
  • RAK WisBlock 4631 — modular platform, popular for fixed site installs
  • Seeed WIO Tracker / T1000-E — compact tracker with GPS

Check the MeshCore project site for the current list of supported hardware and recommended configurations.

Step 2 — Flash MeshCore firmware

Unlike Meshtastic, MeshCore firmware images are specific to the node role you intend to run. Common roles include:

  • Repeater — fixed infrastructure node that relays traffic across the mesh
  • Room Server — acts as a message store for group chat rooms
  • Client — personal node used with the companion app

Flash the appropriate firmware for your board using the MeshCore web flasher or by following the instructions in the MeshCore documentation.

Step 3 — Install the MeshCore companion app

The MeshCore companion app connects to your node over Bluetooth (or serial) and gives you access to messaging, node management, and configuration.

  • Android: available on Google Play (search "MeshCore")
  • iOS: available on the App Store
  • Desktop: a browser-based companion interface is also available for serial-connected nodes

Open the app, pair to your node, and set your callsign or display name.

Step 4 — Configure CNYmesh radio settings

All CNYmesh MeshCore nodes use a shared radio configuration so that they can hear and communicate with each other. Apply these settings to your node using the companion app or serial console.

Base Radio Configuration

  • Frequency 910.525 MHz
  • Bandwidth 62.5 kHz
  • Spreading Factor 7
  • Coding Rate 8
  • TX Power 22

Apply via serial console

  • set freq 910525
  • set radio bw 62500 sf 7 cr 8
  • set tx 22

Verify settings

  • get radio
  • get freq
  • get tx

Step 5 — Enable 2-byte hash mode

MeshCore supports a 2-byte path hash mode that reduces ID collisions as the network grows. All CNYmesh repeaters and infrastructure nodes should have this enabled.

Apply 2-Byte Hash Mode

  • set path.hash.mode 1
  • reboot
  • clock sync

Verify after reboot

  • get path.hash.mode

After rebooting, check the device status tab to confirm uptime and clock sync.

Step 6 — Configure regional channels

CNYmesh uses a nested regional structure to keep local traffic organized and prevent unnecessary propagation into neighboring mesh areas.

Regional Structure

  • us — national level
  • us-ny — New York State level
  • us-ny-cny — Central New York local level

Apply regional configuration

  • region put us
  • region put us-ny us
  • region put us-ny-cny us-ny
  • region allowf us
  • region allowf us-ny
  • region allowf us-ny-cny
  • region save

Verify

  • region

The output should show the full nested hierarchy from us → us-ny → us-ny-cny.

Step 7 — Go on the air

With radio settings and regional configuration in place, your node is ready to join the CNYmesh MeshCore network. A few things to keep in mind:

  • Fixed infrastructure nodes should be elevated and powered 24/7 for best coverage
  • Weatherproof enclosures are strongly recommended for outdoor installs given CNY winters
  • Coordinate with the community on Discord to find coverage gaps worth filling
  • Check the coverage map and join our channels to stay current on deployments
